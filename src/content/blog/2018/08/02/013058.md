なんで依存足りてないの〜的なやつです。

![f:id:treeapps:20180802010416p:plain](/hatena-images/images/fotolife/t/treeapps/20180802/20180802010416.png)

最近kotlin + Spring boot2(webflux)をやっているのですが、slf4jでログ出力すると、以下のログが出ていました。

```sh
2018-08-01 01:48:29,474 INFO  [reactor-http-nio-3] [io.netty.util.internal.logging.Slf4JLogger:101] Starting without optional epoll library
2018-08-01 01:48:29,475 INFO  [reactor-http-nio-3] [io.netty.util.internal.logging.Slf4JLogger:101] Starting without optional kqueue library
```

これを見て「はて〜〜〜？」と思ったので、少し調べてみました。

- [環境](#環境)
- [原因](#原因)
- [依存が足りていない？](#依存が足りていない)

<a id="環境"></a>

### 環境

   OS macOS   フレームワーク Spring boot v2.0.3.RELEASE   APサーバ netty

要はmac上でSpring boot2(webflux)をnettyで動かしているだけです。

<a id="原因"></a>

### 原因

いきなり結論ですが、gradleに以下の依存が足りないため出力されていました。

```groovy
compoile("io.netty:netty-transport-native-kqueue:4.1.28.Final")
```

<a id="依存が足りていない"></a>

### 依存が足りていない？

./gradlew dependenciesで依存jarの中から、「netty-transport-native」に関するものを探ってみると、「spring-boot-starter-webflux」を入れると、自動的にepollとunix-commonが入るようです。

```sh
+--- org.springframework.boot:spring-boot-starter-webflux:2.0.3.RELEASE
|    +--- org.springframework.boot:spring-boot-starter-reactor-netty:2.0.3.RELEASE
|    |    \--- io.projectreactor.ipc:reactor-netty:0.7.8.RELEASE
|    |         +--- io.netty:netty-transport-native-epoll:4.1.25.Final
|    |         |    +--- io.netty:netty-transport-native-unix-common:4.1.25.Final
```

![f:id:treeapps:20180802012152p:plain](/hatena-images/images/fotolife/t/treeapps/20180802/20180802012152.png)

・・・・あれ？

macってFreeBSDを魔改造したOSだから、netty-transport-native-unix-commonの依存があればよさそうでは？と思ったのですが、どうやらnetty-transport-native-unix-commonだけでは駄目みたいです。

となるとBSD系のkqueueがあればいいかな？という事でnetty-transport-native-kqueueの依存を追加したところ、無事Starting without optional xxx libraryは表示されなくなりました。最初からkqueueも入れておいてくれよー、と思いましたが、macはオプショナル的な扱いでlinuxを標準としているから入っていないのでしょうね。

結果的にepollとkqueueの両方の依存が入る事になりますが、io.netty.util.internal.logging.Slf4JLogger実行時に、OSをもとにepollとkqueueのどちらのjarが利用可能かを判断してくれるので、linuxにデプロイした際は自動的にepollが参照され、macで実行した際はkqueueが参照されます。

## 準備するもの

- テンプレート画像
- Image Magick
- フォントファイル
- ブログタイトル
- コマンド

### テンプレート画像

![X Cardテンプレート](/2026/04/05/x-post-image-template.png)

macOSなら、無料化された [Affinity](https://www.affinity.studio/ja_jp) や [Gimp (OSS)](https://www.gimp.org/) 等で作成してください。

上手くやれば `ChatGPT` や `Gemini` に作成させることもできそうです。

### Image Magick

[Image Magick](https://imagemagick.org/#gsc.tab=0) が必要です。

macOSなら [HomebrewでImageMagickをインストール](https://formulae.brew.sh/formula/imagemagick) でインストールしてください。

### フォントファイル

今回私は等幅で美麗な [PlemolJP](https://github.com/yuru7/PlemolJP) を使っています。

macOSなら [HomebrewでPlemolJPをインストール](https://github.com/yuru7/PlemolJP/blob/main/doc/install_via_homebrew.md) してください。

ここからが重要です。macOSの場合、Homebrewでインストールした `PlemolJP` は `~/Library/Fonts/` にあります。

```
❯ cd ~/Library/Fonts/
❯ ll PlemolJP* | head -5
Permissions Size User   Group Date Modified Name
.rw-r--r--@ 5.6M tester admin 29 Jun  2025  PlemolJP-Bold.ttf
.rw-r--r--@ 6.0M tester admin 29 Jun  2025  PlemolJP-BoldItalic.ttf
.rw-r--r--@ 5.6M tester admin 29 Jun  2025  PlemolJP-ExtraLight.ttf
.rw-r--r--@ 6.0M tester admin 29 Jun  2025  PlemolJP-ExtraLightItalic.ttf
```

今回は太字の `PlemolJP-Bold.ttf` を使います。

例えば以下のように `.ttf` ファイルをコピーします。

```shell
cp ~/Library/Fonts/PlemolJP-Bold.ttf ~/project/fonts
```

### ブログタイトル

今回は `ローカルLLM＋Codex CLIとCodex.appを同居させる` とします。

### コマンド

まずはローカル環境で以下のコマンドを実行します。

テンプレート画像の `./x-post-image-template.png` とフォントの `./fonts/PlemolJP-Bold.ttf` のパスは任意に変更してください。

```shell
magick ./x-post-image-template.png \
  \( -background none \
     -fill black \
     -font "./fonts/PlemolJP-Bold.ttf" \
     -gravity northwest \
     -interline-spacing 0 \
     -size 910x300 \
     caption:"ローカルLLM＋Codex CLIとCodex.appを同居させる" \) \
  -gravity northwest \
  -geometry +141+129 \
  -composite \
  output.png
```

上記を実行すると、 `output.png` と出力します。結果は以下の通りです。

![焼き込み後のX Card](/2026/04/05/output.png)

これでテンプレート画像に `Image Magick` でブログタイトルを焼き込むことができました 👏👏👏

## このブログではどうしてるの？

この作業を `ローカル環境` で行うのは大変というか、面倒というか、作業を忘れそうですね。

当ブログでは以下を `GitHub Actionsで自動化` しており、具体的には以下を行っています。

- `package.json` に [ブログカード生成スクリプト](https://github.com/treetips/blog/blob/e2c6c612ec144e8d86a7676c51ddd8a630a0e5f1/package.json#L16) を追加。
- [TypeScriptで Image Magick のコマンドを実行できるよう実装](https://github.com/treetips/blog/blob/e2c6c612ec144e8d86a7676c51ddd8a630a0e5f1/scripts/generate-x-card-images.ts) 。
- `GitHub Actions` で以下を行っています。
  - GitHub ActionsのUbuntuに [Image Magick をインストール](https://github.com/treetips/blog/blob/62fba92ba611e14a0d958e61493ea0e20419acb7/.github/workflows/deploy.yml#L42-L43) するコマンドを追加。
  - GitHub Actionsに [bun run x-card](https://github.com/treetips/blog/blob/62fba92ba611e14a0d958e61493ea0e20419acb7/.github/workflows/deploy.yml#L45-L49) でブログカードを動的に生成するコマンドを追加。このスクリプトは `生成した画像パスを記事markdownに反映` も含みます。
  - [ブログカード画像をgit push](https://github.com/treetips/blog/blob/62fba92ba611e14a0d958e61493ea0e20419acb7/.github/workflows/deploy.yml#L54-L70) 。


このように自動化しました。

## Q and A

### Q. え？macOSのフォントファイルをLinuxであるUbuntuでも使えるの？

A. 使えました。

### Q. 実装作業大変ですねぇ・・・

A. [Codex.app](https://openai.com/ja-JP/codex/get-started/) は優秀でした。

### Q. Image Magick でよく狙った位置にテキストを配置できましたね？

A. ・・・大変でした。こればかりはCodexで一発で実装することはできず、何度も何度もCodexに指示をし直し、ようやく狙った位置にテキスト書き込むことができました。

### Q. Xカードのテンプレート画像、どうやって作った？

A. [Affinity](https://www.affinity.studio/ja_jp) で作りました。はっきりいって・・・・ `このテンプレート画像を作る作業が一番時間がかかりました` 。

画像アプリが多機能過ぎて簡単な機能を探すのも時間がかかり、↓たかがこの程度の画像作るだけで時間が溶けていきました。

![X Cardテンプレート](/2026/04/05/x-post-image-template.png)

### Q. あれ？Image MagickはLinuxではconvertコマンドだよね？

A. はい。macOSでインストールすると `magick` コマンドで、Linuxでインストールすると `convert` コマンドです。

なので、 `ローカル環境の場合はmagickを、GitHub Actionsの場合はconvertを` というコマンド名の分岐を実装する必要があります。

はい。Codexさんは優秀なので簡単に実装してくれました。

### モバイルファーストなフレームワーク

<a id="ionic"></a>

#### ionic

[Build Amazing Native Apps and Progressive Web Apps with Ionic Framework and Angular](http://ionicframework.com/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://ionicframework.com/\)](http://b.hatena.ne.jp/entry/http://ionicframework.com/) iOS7のUIに倣ったフレームワークです。非常に綺麗なUIです。 ![f:id:treeapps:20131226233919p:plain](/hatena-images/images/fotolife/t/treeapps/20131226/20131226233919.png)

<a id="良い点"></a>

##### 良い点

- 数少ないiOS7風のUI。
- UIが綺麗。

<a id="悪い点"></a>

##### 悪い点

- 圧倒的なドキュメント不足。
- 圧倒的なサンプル不足。
- angular.jsの使用を強制される。
- 覚えるまでにかなり時間がかかる。

<a id="ChocolateChip-UI"></a>

#### ChocolateChip-UI

[ChocolateChip UI](http://www.chocolatechip-ui.com/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://www.chocolatechip-ui.com/\)](http://b.hatena.ne.jp/entry/http://www.chocolatechip-ui.com/) iOS7のOS自体のUIに倣ったフレームワークです。 iOS7、andriod、metroのテーマに切り替えできます。 ![f:id:treeapps:20131226234621p:plain](/hatena-images/images/fotolife/t/treeapps/20131226/20131226234621.png)

<a id="良い点-1"></a>

##### 良い点

- 今のところiOS風のUIはこれだけです。
- アニメーションも豊富。

<a id="悪い点-1"></a>

##### 悪い点

- あまりにも重厚過ぎる。
- js回りの機能が豊富過ぎて調査する気が失せる。

<a id="CLANK"></a>

#### CLANK

[My blog &ndash; Just another WordPress site](http://getclank.com/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://getclank.com/\)](http://b.hatena.ne.jp/entry/http://getclank.com/) フラットなUIを持つフレームワークです。 ![f:id:treeapps:20131226235207p:plain](/hatena-images/images/fotolife/t/treeapps/20131226/20131226235207.png)

<a id="良い点-2"></a>

##### 良い点

- UIは綺麗にまとまっている。
- 他のFWとくらべて機能的に軽量。

<a id="悪い点-2"></a>

##### 悪い点

- 微妙にデザイン崩れしてる（↑の画像のボトムタブバーの部分とか）
- 機能はそれ程多くない。

<a id="RATCHET"></a>

#### RATCHET

http://maker.github.io/ratchet/[!\[\]\(//b.hatena.ne.jp/entry/image/http://maker.github.io/ratchet/\)](http://b.hatena.ne.jp/entry/http://maker.github.io/ratchet/) 元々iPhoneのプロトタイプを作成するために作られたFWのようです。 ![f:id:treeapps:20131227000357p:plain](/hatena-images/images/fotolife/t/treeapps/20131227/20131227000357.png)

<a id="良い点-3"></a>

##### 良い点

- UIは完全にiOSになっており、非常にクオリティが高く美しいUI。
- jsが1ファイル、cssが1ファイル、画像が3ファイル、と超軽量。

<a id="悪い点-3"></a>

##### 悪い点

- iOS6以前のUIなのが残念！！iOS7だったら最高でした。
- プロトタイプ作成用という事で、コンポーネントはリッチな挙動をしません。

<a id="Fries"></a>

#### Fries

[UGG en la tienda online de Anelai - Getfri.es](http://getfri.es/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://getfri.es/\)](http://b.hatena.ne.jp/entry/http://getfri.es/) androidのUIを持つFWです。 ![f:id:treeapps:20131227004538p:plain](/hatena-images/images/fotolife/t/treeapps/20131227/20131227004538.png)

<a id="良い点-4"></a>

##### 良い点

- まともなandroidのUIになっている。
- ちゃんとアニメーションする。

<a id="悪い点-4"></a>

##### 悪い点

- androidのUI自体好みが分かれそう。
- コンポーネントは少なめ。

<a id="Foundation"></a>

#### Foundation

[The most advanced responsive front-end framework in the world. | Foundation](http://foundation.zurb.com/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://foundation.zurb.com/\)](http://b.hatena.ne.jp/entry/http://foundation.zurb.com/) モバイルに特化したフラットUIの多機能フレームワークです。 ![f:id:treeapps:20131227001520p:plain](/hatena-images/images/fotolife/t/treeapps/20131227/20131227001520.png)

<a id="良い点-5"></a>

##### 良い点

- 他のFWと一線を画するレベルでしっかり作られている。
- モバイル特化FWでありながら、bootstrap並にコンポーネントがある。

<a id="悪い点-5"></a>

##### 悪い点

- 個人的にボトムのタブバーが欲しかった。

小並感

Foundationだけ作り込みのレベルが違う印象を受けました。 ChocolateChip-UIはかなりいい感じですが、重厚すぎて覚えたく無い感じです。

他のFWは全体的に中途半端です。妙なデザイン崩れ、コンポーネントの少なさ、ドキュメントの少なさ、サンプルの少なさ、全てに於いて中途半端さが目立ちました。更に、angular.jsを強要されたり、重厚な専用jsを強要されるのも沢山あります。

これらを踏まえてどれを使うかを迷っているのですが、FoundationかCLANKかRATCHETが候補になりそうです。

<a id="おまけレスポンシブフレームワーク"></a>

### おまけ：レスポンシブフレームワーク

※ グリッドのみのレスポンシブフレームワークは除外します。

<a id="twitter-bootstrap"></a>

#### twitter bootstrap

[Bootstrap ・ The most popular HTML, CSS, and JS library in the world.](http://getbootstrap.com/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://getbootstrap.com/\)](http://b.hatena.ne.jp/entry/http://getbootstrap.com/) 皆大好きbootstrap。twitter製で豊富でリッチなコンポーネントを持ち、非常に使い勝手がよいです。 モバイル向きのUIでは無いです。

<a id="Pure"></a>

#### Pure

[Pure](http://purecss.io/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://purecss.io/\)](http://b.hatena.ne.jp/entry/http://purecss.io/) Yahoo製の超軽量FWです。jsが一切無いピュアcssで作られています。

<a id="Topcoat"></a>

#### Topcoat

[Topcoat](http://topcoat.io/topcoat/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://topcoat.io/topcoat/\)](http://b.hatena.ne.jp/entry/http://topcoat.io/topcoat/) Adobe製の超高速FWです。まるでPhotoshopのUIのようです。 速さが売りらしく、ベンチマークを載せています。

<a id="Kraken"></a>

#### Kraken

[Kraken](http://cferdinandi.github.io/kraken/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://cferdinandi.github.io/kraken/\)](http://b.hatena.ne.jp/entry/http://cferdinandi.github.io/kraken/) 軽量なフラットUIのFWです。極力軽量だけどコンポーネントは欲しい！という方にいいかもしれません。

<a id="Cascade"></a>

#### Cascade

[Cascade Framework](http://jslegers.github.io/cascadeframework/)[!\[\]\(//b.hatena.ne.jp/entry/image/http://jslegers.github.io/cascadeframework/\)](http://b.hatena.ne.jp/entry/http://jslegers.github.io/cascadeframework/) ちょっとコンポーネントが少ないフラットUIのFWです。 モバイルの事を考えなければ、ドキュメントが多いbootstrapを使っていればいいかと思います。 他のFWはちょっと中途半端感が拭えないので、選択肢にはならないかも？？

このデータ垂れ流しサイトとは、csvやweb apiで取得したデータしかコンテンツが無いサイトの事です。 例えば、amazon product advertising apiで商品情報を取得・表示をしているだけのサイトであったり、商品情報のcsvを毎日ftpして貰い、それを取り込んでそのまま表示しているだけのサイトの事です。

これは個人サイトより企業サイトで頻繁に目にします。

<a id="データ垂れ流しサイトの問題"></a>

### データ垂れ流しサイトの問題

<a id="コンテンツの所有者は誰か"></a>

#### コンテンツの所有者は誰か

当然あなたはそのコンテンツの所有者ではありません。従って他人のコンテンツを自分のサイトに表示しても、所有者のサイトには絶対に勝てません。コンテンツの表示レイアウトを変えてもダメです。google先生は全てお見通しです。恐らくですが、コンテンツを形態素解析し、単語にバラしてから所有者サイトとのマッチ度を比較したりしているでしょう。もしそうなら、取得したコンテンツの「である調」を「ですます調」に置換するような小細工も簡単に看破されます。改行なんかも無駄ですね。

<a id="垂れ流しデータ以外に何がある"></a>

#### 垂れ流しデータ以外に何がある？

例えば、データ所有者のサイトには「データの検索機能が無い」とします。しかしあなたのサイトに「データの検索機能が有る」とします。その場合、それは付加価値なので、googleから評価されます。

しかし所有者のサイトにもあなたのサイトにも検索機能があったら・・・？あなたのサイトの検索機能に付加価値はありません。

あなたのサイトには所有者のサイトには無い付加価値はありますか？ブログパーツ等はダメです。そもそもブログパーツも他者の機能・コンテンツを借りているだけです。どうですか？何かあなたのサイトにしかないコンテンツはありますか？

<a id="データ垂れ流しサイトがやるべき事"></a>

### データ垂れ流しサイトがやるべき事

<a id="オリジナルコンテンツを作る"></a>

#### オリジナルコンテンツを作る

実に単純です。あなたのサイトにオリジナルコンテンツが有れば、それだけでいいのです。例えばコメント機能を追加して、ユーザにコメントして貰うだけでもOK。勿論そのコメント数が1件だけだったら、それは付加価値と呼べないのでNG。

他にも、取得したデータについての感想やレビューを必ず付加し、それを表示するのもOK。企業サイトであれば管理画面を作り、スタッフによるコメント・レビュー投稿機能を付け、画面に表示するのが良いでしょう。できるだけ多くの文章を書いて貰えば、所有者サイトとの類似度は下がり、あなたのサイトの価値が上がるでしょう。

自分達でコンテンツが生み出せる場合は生み出し、厳しいならユーザに協力を求める（コメント機能とか）と良いでしょう。

<a id="所有者サイトを上回る機能を作る"></a>

#### 所有者サイトを上回る機能を作る

所有者サイトにデータの検索機能があるとして、それを上回る機能を作りましょう。例えば、データに住所があるなら、住所をジオコーディングして緯度経度を抽出し、solrやelasticsearchを使って位置情報検索できるようにするなど、他人が生み出しにくい機能を追加する事も重要です。

<a id="重要な心構え"></a>

#### 重要な心構え

重要なのはSEOではない点です。SEOはSearchEngineOptimize、つまり検索エンジンに対する施策ではオリジナルコンテンツにはならない点を考慮しましょう。

例えば、取得データの周りに、別の取得データを使って機械的なリンクを動的に生成して「対策した！」というケースを多く見ます。これはNGです。これはクローラが辿れるリンクを増やすだけの施策で、ユーザにメリットの無い邪魔なリンクである場合が多いです。SEO業者はよくこういった「動的リンクの追加」を提案して来ますが、これは付加価値を産みません。

あくまでユーザにとってメリットとなるオリジナルコンテンツの追加である必要があります。断じて検索エンジン向けであってはいけません。

この説明から解る通り、オリジナルコンテンツを生み出す事は大変な労力を要するのです。まあ当然ですね。

そもそもデータの所有者はそれだけ苦労してそのデータを作っているわけで、あなたもそれと同様な労力が必要になります。それができるサイトが高品質であり、ブログパーツやSNSボタンをペタペタはってオリジナルコンテンツを気取るサイトは低品質サイトとなります。

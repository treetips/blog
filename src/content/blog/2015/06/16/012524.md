<a id="現象"></a>

### 現象

   名称 バージョン   Intellij IDEA 14   JDK 1.8   Gradle 2.4   spring-boot-gradle-plugin 1.3.0.BUILD-SNAPSHOT   springloaded 1.2.4.BUILD-SNAPSHOT   spring-boot-starter-web 1.2.3.RELEASE   spring-boot-starter-test 1.2.3.RELEASE

この環境でspring loadedが動きませんでした。

設定はオフィシャルのドキュメント↓を参考にしていました。 [86. Hot Swapping](http://docs.spring.io/spring-boot/docs/current/reference/html/howto-hotswapping.html#howto-reload-springloaded-gradle-and-intellij)

spring boot の起動は以下のように、Controllerから右クリックでRunしていました。 ![f:id:treeapps:20150616005215p:plain](/hatena-images/images/fotolife/t/treeapps/20150616/20150616005215.png)

なんだこりゃ。普通にspring boot起動してるのに、何故かreloadされないな。eclipseだと全くなにもせず起動してるんだけどなあ

<a id="必須の設定を確認する"></a>

#### 必須の設定を確認する

<a id="Intellij-IDEAとgradleのclassファイルの出力先が違うので合わせる"></a>

##### Intellij IDEAとgradleのclassファイルの出力先が違うので合わせる

これはもう有名で、以下を設定する必要があります。 [86. Hot Swapping](http://docs.spring.io/spring-boot/docs/current/reference/html/howto-hotswapping.html#howto-reload-springloaded-gradle-and-intellij)

<a id="自動ビルドを有効にする"></a>

##### 自動ビルドを有効にする

preferencies -> Build Excution, Deployment -> Compiler -> Make project automatically にチェックを付けます。 ![f:id:treeapps:20150616010000p:plain](/hatena-images/images/fotolife/t/treeapps/20150616/20150616010000.png) 設定としてはこれだけのはず。しかし何故かリロードされず・・・

<a id="原因判明"></a>

### 原因判明

やるお、こういう時は落ち着いてお茶でも飲んでもう一度設定をよく見てみよう。 そうだな。こういう時は大抵しょうもない罠があるもんね。よく見てみるよ。![f:id:treeapps:20150616010347p:plain](/hatena-images/images/fotolife/t/treeapps/20150616/20150616010347.png) ふんふんふーん♪ラーメンうめー♪・・・・ん？![f:id:treeapps:20150616010835p:plain](/hatena-images/images/fotolife/t/treeapps/20150616/20150616010835.png) ・・・・・

only works while not running / debugging RunやDebugの実行中は自動ビルドは無効だよ〜ん♪テヘ☆

貴様か！！

はい。こいつが罠でした。最初にHelloControllerをIDEA上から起動していましたが、これが誤りです。IDEAから起動してしまうと、起動中は自動コンパイルが無効になってしまうので、spring loadedがうんともすんとも言わない訳でした。。

<a id="解決策"></a>

### 解決策

IDEAからbootRunするのではなく、普通に以下のようにコンソールからgradle bootRunすればいいのでした。 ![f:id:treeapps:20150616011908p:plain](/hatena-images/images/fotolife/t/treeapps/20150616/20150616011908.png) これでばっちりspring loadedが有効になってくれます。

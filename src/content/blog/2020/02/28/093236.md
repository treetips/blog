<a id="staticが見つからない"></a>

### staticが見つからない

ビルドやらcopyやらをし、最終的に npx cap open electronすると、以下のようになりました。

![f:id:treeapps:20200228090450p:plain](/hatena-images/images/fotolife/t/treeapps/20200228/20200228090450.png)

```text
GET file:///static/css/10.4c99b71.chunk.css index.html:1
net::ERR_FILE_NOT_FOUND
```

githubに以下のissueが挙がっていました。

[Better solution for electron <base> url \(or just document it\) · Issue #639 · ionic-team/capacitor](https://github.com/ionic-team/capacitor/issues/639#issuecomment-496547899)

<a id="対応策"></a>

### 対応策

<a id="publicindexhtml"></a>

#### public/index.html

以下をコメントアウトし、hrefに動的に値が入るように %PUBLIC_URL% を指定します。

```html
<base href="/" />
```

↓

```html
<!-- <base href="/" /> -->
<base href="%PUBLIC_URL%/">
```

<a id="補足"></a>

##### 補足

githubのissueではコメントアウトすれば動く旨が記述されており、確かにコメントアウトすれば動きます。

しかし、ボタンをクリックしたらalertコンポーネントが表示されるコードでelectronを実行すると以下のエラーが発生しました。

```txt
cannot read property 'isproxied' of undefined
```

上記エラーを解消するため、baseのhrefを以下にしたところ、解消しました。

```html
<base href="%PUBLIC_URL%/">
```

<a id="packagejson"></a>

#### package.json

以下を追加します。追加位置はどこでもいいので、今回は末尾に追加しています。

```json
  "description": "An Ionic project"
}
```

↓

```json
  "description": "An Ionic project",
  "homepage": "."
}
```

<a id="ビルドする"></a>

#### ビルドする

```sh
ionic build
npx cap copy
npx cap open electron
```

さて、結果は・・・

![f:id:treeapps:20200228091607p:plain](/hatena-images/images/fotolife/t/treeapps/20200228/20200228091607.png)

OKです。

修正前は

```html
<link href="/static/xxxx" rel="stylesheet">
```

だったのが以下のように「.」始まりに変わり、解決したようです。

```html
<link href="./static/xxxx" rel="stylesheet">
```

👍
